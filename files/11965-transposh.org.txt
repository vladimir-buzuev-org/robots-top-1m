<p class="entry-meta"><time class="entry-time" itemprop="datePublished" datetime="2016-05-01T23:33:44+00:00">May 1, 2016</time> by <span class="entry-author" itemprop="author" itemscope itemtype="https://schema.org/Person"><a href="http://transposh.org/author/ofer/" class="entry-author-link" itemprop="url" rel="author"><span class="entry-author-name" itemprop="name">ofer</span></a></span> <span class="entry-comments-link"><a href="http://transposh.org/version-0-9-9-0-workers-of-the-world-unite/#comments">8 Comments</a></span> </p></header><div class="entry-content" itemprop="text"><p>For the first of May, the international workers day we have finally released the very long overdue and anticipated version of 0.9.9.0.</p>
<p>What is this version good for?</p>
<p>First &#8211; support for <a href="http://baidu.cn">Baidu</a>, the Chinese search giant which also have a translation engine, with the support for Cantonese which is rather unique for this engine. Another good thing is that with this engine, websites that are hosted in China can get access to a working translation engine.</p>
<p>Second &#8211; we have fixed the current key less support of both Google and Yandex, so actually they should work again now. So if a few pages in your site were not translated, the new version got you covered. A few languages were added in the process.</p>
<p>Third &#8211; lots of changes, additions and bug fixes all around. Yeah, we need to fix the documentation and FAQ for some.</p>
<p>Why are you still reading this? Just upgrade today, and work tomorrow!</p>

<p class="entry-meta"><time class="entry-time" itemprop="datePublished" datetime="2015-10-28T00:01:15+00:00">October 28, 2015</time> by <span class="entry-author" itemprop="author" itemscope itemtype="https://schema.org/Person"><a href="http://transposh.org/author/ofer/" class="entry-author-link" itemprop="url" rel="author"><span class="entry-author-name" itemprop="name">ofer</span></a></span> <span class="entry-comments-link"><a href="http://transposh.org/version-0-9-8-0-welcome-yandex/#comments">1 Comment</a></span> </p></header><div class="entry-content" itemprop="text"><p>There is more to the world than just Google Translate, there is also more to the world than just Bing Translate, apparently the ~150 million Russians are also doing great stuff, with Yandex, the most popular search engine in Russia.</p>
<p>Yandex supports many languages and produces different results than the other engines, it also specializes in languages that are spoken inside Russia such as Bashkir, Kirghiz and Tatar. Which we were now able to add to the list of languages, making the total number of supported auto-translated languages to a whopping 95!</p>
<p>The addition of Yandex also triggered changed in the way we deploy auto translations, so it is now possible to set the preference of the translation engine you would like to use.</p>
<p>We also had many (and we mean MANY) bugs fixed in this release, the change log will give you a good taste of this:</p>
<ul>
<li>Yandex translator support added</li>
<li>Added the option to choose preferred translators order via drag and drop</li>
<li>Fix critical bug in translate post and translate on publish</li>
<li>Fixes in language selection UI (long language names)</li>
<li>Fix translation dialog not loading on first click</li>
<li>No longer expose Google keys externally</li>
<li>Fix the clean automatic translations in the utils tab</li>
<li>Support Bashkir, Kyrghiz and Tatar thanks to Yandex &#8211; Now supporting 95 Languages!</li>
<li>Added Swahili support to Bing</li>
<li>Reduced wait time between posts in translate all to 2 seconds</li>
<li>Fix shortcodes when wrapping a &lt;p&gt; tag</li>
<li>New Google icon</li>
<li>Fix WP_Widget deprecation call for wordpress 4.3</li>
<li>Minor fix for buddypress search integration</li>
<li>Fix some cases when https urls would not be rewritten</li>
<li>Fix rewrite support for three letter languages (currently only ceb)</li>
</ul>
<p>Enjoy this version!</p>

<p class="entry-meta"><time class="entry-time" itemprop="datePublished" datetime="2015-07-29T01:34:37+00:00">July 29, 2015</time> by <span class="entry-author" itemprop="author" itemscope itemtype="https://schema.org/Person"><a href="http://transposh.org/author/ofer/" class="entry-author-link" itemprop="url" rel="author"><span class="entry-author-name" itemprop="name">ofer</span></a></span> <span class="entry-comments-link"><a href="http://transposh.org/version-0-9-7-2-shortened-shortcodes/#respond">Leave a Comment</a></span> </p></header><div class="entry-content" itemprop="text"><p>The release train has finally picked up some speed, so we are proud to present this new version.<br />
This version fixes the Google proxy again, this is happening because there are things that we really cannot control. But again, it works.</p>
<p>The other change is some fixes in shortcodes, you can read the much improved documentation in the following link:<br />
<a href="http://trac.transposh.org/wiki/ShortCodes">Shortcode support in Transposh wordpress plugin</a></p>
<p>The main changes include the additional <code>[tpe]</code> shortcode which was added on top of the older <code>[]</code>.<br />
This code is used for self enclosing codes, where it is not necessary to close the tag, this is because a caveat in wordpress which causes<br />
the code parsing to go berserk when mixing self-enclosed and closing shortcodes.</p>
<p>Also, there is no need to specify <code>="y"</code> any more, so now <code>[tpe mylang]</code> will simply work and output: <code>en</code></p>
<p>We urge you to upgrade, and enjoy this new version.</p>

<p class="entry-meta"><time class="entry-time" itemprop="datePublished" datetime="2015-06-06T00:00:48+00:00">June 6, 2015</time> by <span class="entry-author" itemprop="author" itemscope itemtype="https://schema.org/Person"><a href="http://transposh.org/author/ofer/" class="entry-author-link" itemprop="url" rel="author"><span class="entry-author-name" itemprop="name">ofer</span></a></span> <span class="entry-comments-link"><a href="http://transposh.org/version-0-9-7-0-fixes-and-features-all-around-and-superproxy-alpha/#comments">2 Comments</a></span> </p></header><div class="entry-content" itemprop="text"><p>New version is here!</p>
<p>Please upgrade! Please, Please upgrade! Please with some sugar on top.</p>
<h4>Why you should upgrade?</h4>
<ul>
<li>This version fixes the Google proxy support, which means that the automated Google Translation will work again.</li>
<li>This version updates the database format, which means wordpress 4.2 is supported, and you are expected to save ~40% of the current table size</li>
<li>We finally have an initial version of a backend translation editor, it will show you the strings real people have translated and allow you to quickly review and delete those.</li>
</ul>
<h4>How should you upgrade?</h4>
<p>With some luck, you have the option to simply upgrade in your wordpress dashboard. It is recommended that you backup, as there&#8217;s no way back after the database upgrade has completed.</p>
<p>Any quirks, issues, problems &#8211; the <a href="http://transposh.org/contact-us/">contact us form</a> is your friend.</p>
<h4>What is that SuperProxy and why should you enable this?</h4>
<p>SuperProxy is our new &#8220;Get rich quick&#8221; scheme. hmm&#8230;</p>
<p>Actually, we are using the end nodes to allow companies and individuals to have distributed access to the web. We will be paying for this traffic, so if you bought a hosting service and you have tons of data transfer you are not using, we will really appreciate that you enable this feature, <em><strong>it will support us and will probably be worth your while</strong></em>. We will publish more details about this as they become available.</p>
<p>And &#8211; to the rest of the changelog:</p>
<ul>
<li>Woo integration fixes</li>
<li>Ditch the base64 encoding and reduce size of translatable pages</li>
<li>Widget fixes makes flags show correctly when wrong box sizing was used</li>
<li>Removal of front end progress bar, it was rarely seen or used and caused more trouble than it was worth</li>
<li>3 new languages supported by bing</li>
<li>locale shortcode en_US</li>
<li>avoid translation of non text/json content fixes issues with some download addon plugins</li>
<li>Support the placeholder attribute, thanks Mark Serellis</li>
<li>Some support to the business directory plugin</li>
</ul>
<p>So &#8211; Enjoy this version, and don&#8217;t be a stranger, let us know how it works for you.</p>
